# 프로젝트 작업 가이드

## 목적
- 본 프로젝트는 입금 확인 요청/조회 흐름을 안정적으로 제공하는 것이 최우선이다.

## 코드 원칙
- UI 문구는 한국어 사용자 기준으로 간결하고 자연스럽게 작성한다.
- 네이밍은 기능 중심으로 작성하고, 임시/자동 생성 느낌의 이름은 사용하지 않는다.
- 변경은 최소 범위로 적용하고 기존 동작을 깨지 않도록 한다.

## UI 원칙
- 모바일 우선으로 구성하고 입력/조회 흐름을 한 번에 이해할 수 있게 배치한다.
- 주요 액션 버튼은 한 화면에 하나의 우선순위를 갖도록 유지한다.

## 운영 원칙
- 민감정보(비밀번호/키)는 코드에 하드코딩하지 않고 환경변수로 관리한다.
- 운영진 기능(상태 변경/일괄 삭제)은 로그가 남도록 유지한다.
